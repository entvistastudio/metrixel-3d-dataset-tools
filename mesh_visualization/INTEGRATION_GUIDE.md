# Metrixel Mesh Visualization Integration Guide

This guide explains how the mesh visualization tools integrate with Metrixel and how to use or bundle them.

## 🎯 Integration Overview

The mesh visualization Python scripts in `tools/mesh_visualization/` are **open to use without restriction**. You can run them directly from the tools directory (e.g. from the command line or from Metrixel) for both single-frame viewing and animation; no subscription or tier limits apply.

## 📁 File Structure

```
metrixel/
└── tools/
    └── mesh_visualization/
        ├── visualize_mesh_metrixel.py    # Single frame viewer
        ├── animate_mesh_metrixel.py      # Animation viewer

        ├── config.py                         # Configuration
        ├── requirements.txt                  # Python dependencies
        ├── setup.sh                          # Linux/macOS setup
        ├── setup.bat                         # Windows setup

        └── INTEGRATION_GUIDE.md              # This file
```

## 🔧 Technical Integration

### 1. Coordinate System Alignment
The visualization tools use Metrixel's coordinate system:
- **X → Right** (horizontal movement)
- **Y → Up** (vertical movement)
- **Z → Toward** (depth toward camera/viewer)

### 2. File Format Compatibility
- **Input**: `.pt` files (PyTorch tensors) generated by Metrixel
- **Vertices**: `vert_000_000_000_XXXX.pt`
- **Faces**: `face_000_000_000_XXXX.pt`

### 3. Dependencies
- **torch**: For loading `.pt` tensor files
- **matplotlib**: For 3D visualization
- **numpy**: For numerical operations

### 4. Parameter-Based Configuration
The tools use command-line parameters:
- **--mesh_dataset_root**: Path to mesh dataset root (folder containing sequence IDs, e.g. `.../generated/meshes`)
- **--seq**: Sequence ID to visualize
- **--frame**: Specific frame to view (for single frame)
- **--frame_begin/--frame_end**: Frame range (for animation)
- **--fps**: Animation frame rate

## 🛠️ Development Integration

### CMake Integration
Add to Metrixel's CMakeLists.txt:
```cmake
# Copy mesh visualization tools to build directory
install(DIRECTORY tools/mesh_visualization/
    DESTINATION tools/mesh_visualization/
    FILES_MATCHING PATTERN "*.py"
    PATTERN "*.md"
    PATTERN "*.txt"
    PATTERN "*.sh"
    PATTERN "*.bat"
)
```

### Qt Integration
Add menu item in Metrixel UI:
```cpp
// Add to Metrixel menu
QAction* meshVizAction = new QAction("Mesh Visualization", this);
connect(meshVizAction, &QAction::triggered, this, &Metrixel::launchMeshVisualization);
toolsMenu->addAction(meshVizAction);

// Launch mesh visualization with mesh dataset root parameter
void Metrixel::launchMeshVisualization() {
    QString meshRoot = getMeshDatasetRoot();  // e.g. projectPath + "/generated/meshes"
    QStringList args = {
        "--mesh_dataset_root", meshRoot,
        "--seq", "000001",
        "--frame", "000_000_000_0000"
    };
    QProcess::startDetached("python", QStringList() << "visualize_mesh_metrixel.py" << args);
}
```
