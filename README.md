# Metrixel 3D Dataset Visualization Tools

![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)
![Python](https://img.shields.io/badge/Python-3.x-blue)

Utilities for visualizing and inspecting 3D datasets generated by **Metrixel** — both **PyTorch `.pt` mesh tensors** and **signed-distance-field (SDF) volumes**.

These tools let researchers and developers preview geometry, animate mesh sequences, inspect SDF volumes, and debug dataset pipelines used in:

- machine learning
- computer vision
- robotics
- simulation pipelines

Part of the **Metrixel ecosystem** for generating 3D datasets for AI pipelines.

---

## Repository layout

The repo is split into two independent toolsets, each with its own `requirements.txt` and entry-point scripts:

```
metrixel-3d-dataset-tools/
├── mesh_visualization/     # PyTorch .pt mesh tensors — preview + animate
├── sdf_visualization/      # SDF .bin volumes — iso-surface + orthogonal slices
└── examples/               # sample dataset for a quick first run
```

## Mesh Visualization (`mesh_visualization/`)

Load and inspect 3D mesh datasets stored as **PyTorch `.pt` tensors**.

• Load PyTorch `.pt` mesh datasets  
• Visualize mesh geometry in 3D  
• Animate mesh sequences with adjustable FPS  
• Inspect vertices, faces, and mesh structure  
• Gracefully handle missing frames  

```bash
cd mesh_visualization
pip install -r requirements.txt

# Single frame
python visualize_mesh_metrixel.py --mesh_dataset_root /path/to/generated/meshes --seq 000001 --frame 000_000_000_0000

# Animation
python animate_mesh_metrixel.py --mesh_dataset_root /path/to/generated/meshes --seq 000001 --frame_begin 000_000_000_0000 --frame_end 000_000_000_0134 --fps 30
```

## SDF Visualization (`sdf_visualization/`)

Two complementary viewers for the per-iteration `.bin` signed-distance volumes Metrixel writes alongside meshes and images when `--gen_sdf=true`. Each file is `R³` float32 values indexed as `volume[z, y, x]` — negative = inside, positive = outside, zero = surface.

| Tool | What it shows | Best for |
|------|---------------|----------|
| `visualize_sdf_metrixel.py` | Marching-cubes iso-surface, rendered as a 3D mesh | "Did the SDF bake the right shape?" — direct comparison with the source mesh |
| `slice_sdf_metrixel.py` | Three orthogonal heatmap slices (XY/XZ/YZ) with the zero contour | Inspecting interior structure, debugging holes/leaks, sanity-checking at low resolution |

```bash
cd sdf_visualization
pip install -r requirements.txt

# 3D iso-surface (marching cubes)
python visualize_sdf_metrixel.py --sdf_dataset_root /path/to/assets --seq 000001 --frame 000_000_000_0000 --sdf_resolution 16

# Three orthogonal slices through the volume center
python slice_sdf_metrixel.py --sdf_dataset_root /path/to/assets --seq 000001 --frame 000_000_000_0000 --sdf_resolution 16
```

For production training data the recommended resolution floor is `--sdf_resolution=32`; lower values are useful for quick smoke tests. The SDF tools share the mesh tools' coordinate convention (X → right, Y → up, Z → toward camera) so SDF and mesh views render identically.

## Metrixel Integration

These tools are designed to inspect datasets generated by Metrixel, which converts FBX, GLTF/GLB, and OBJ assets into structured datasets including:

• PyTorch mesh tensors (`.pt`)  
• Signed-distance-field volumes (`.bin`)  
• Multi-view render sequences  
• Extracted textures
